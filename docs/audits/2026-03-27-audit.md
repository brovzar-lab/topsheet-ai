## IMMEDIATELY (This Sprint)

### 1. WCAG AA Color Contrast Failures
- **Category:** a11y
- **Impact:** Critical (legal/WCAG compliance)
- **Effort:** S
- **axe-core finding:** 3 nodes fail color contrast (impact: serious)
- **Affected text:** "PRODUCTION PLANNING · POWERED BY AI" tagline (~#757575 on #2A2A2A ≈ 3.8:1, needs 4.5:1), "Your data syncs across all devices" subtitle, "TOPSHEET AI © 2026" footer
- **Fix:** Raise muted text to at least `#8A8A8A` on `#2A2A2A` background, or use `#C8C8C8` (`lemon-text-body`). Replace `text-lemon-gray-500` with `text-lemon-text-muted` in AuthGate subtitle. Footer text (`text-lemon-gray-700`) needs to step up to `text-lemon-gray-500` at minimum.
- **Files:** [src/components/AuthGate.tsx](../../src/components/AuthGate.tsx)

### 2. Auth Screen Missing `<main>` Landmark
- **Category:** a11y
- **Impact:** High (screen reader users cannot navigate to main content)
- **Effort:** S
- **axe-core finding:** `landmark-one-main` violation — auth screen renders no `<main>` element; all content is in divs with no landmark role
- **Fix:** Wrap the sign-in card in `<main className="...">` in AuthGate.tsx. The post-login layout already has `<main>` in App.tsx.
- **Files:** [src/components/AuthGate.tsx](../../src/components/AuthGate.tsx)

### 3. Main JS Bundle 4.1 MB — Mitigation Partially Applied
- **Category:** perf
- **Impact:** High (first-load abandonment on slow connections; >2 MB threshold)
- **Effort:** S (already started)
- **Current state:** `dist/assets/index-pQRbay1M.js` = 4.1 MB (1.3 MB gzip). This is from the pre-optimization build.
- **Mitigation applied:** `vite.config.ts` now has `manualChunks` splitting react, firebase, ai, pdf, excel, and ui libraries into separate chunks. **A new `npm run build` must be run to apply this.**
- **Expected result after rebuild:** Main bundle drops to ~500-800 KB; libraries load in parallel and are cached independently.
- **Files:** [vite.config.ts](../../vite.config.ts) — run `npm run build`

### 4. Series Routes Have No Error Boundary
- **Category:** engineering
- **Impact:** High (crashes in series views = data loss, white screen)
- **Effort:** S
- **Current state:** `ErrorBoundary` wraps only `/project/:id/*` in App.tsx. All `/series/:seriesId/*` routes are unprotected.
- **Fix:** Wrap the series route group in App.tsx with the same `ErrorBoundary` component.
- **Files:** [src/App.tsx](../../src/App.tsx)

```tsx
// App.tsx — wrap series routes identically to project routes
<Route path="/series/:seriesId/*" element={
  <ErrorBoundary>
    <Routes>
      <Route index element={<SeriesDashboardPage />} />
      ...
    </Routes>
  </ErrorBoundary>
} />
```

## LATER (Next 2–4 Sprints)

### 5. No Test Coverage for Pages, Components, or Stores
- **Category:** testing
- **Impact:** High (regressions undetectable; refactoring unsafe)
- **Effort:** L
- **Current state:** 10 unit tests cover lib/ functions only. 0 tests for 17 page components, 8 UI components, 11 Zustand stores, all Firestore operations, and all export generators.
- **12% coverage** (10 test files / 83 source files)
- **Priority test targets:**
  - `budget-store.ts` — centavo math, draft creation/cloning
  - `series-store.ts` — episode management, roster operations
  - `BudgetPage.tsx` — line item editing, section totals display
  - `SchedulePage.tsx` — strip reordering, day assignment
- **Fix:** Use `gsd:add-tests` skill to generate tests for completed phases. Add `data-testid` attributes to all interactive elements first.
- **Files:** All of `src/pages/`, `src/stores/`, `src/components/`

### 6. No E2E Test Suite
- **Category:** testing
- **Impact:** High (critical user workflows — PDF upload → breakdown → budget — are completely untested)
- **Effort:** M
- **Current state:** No Playwright or Cypress configuration. No integration tests.
- **Critical flows to cover:**
  1. Upload PDF → AI breakdown → review elements
  2. Create budget draft → add line items → export Excel
  3. Build schedule → view DOODs
  4. Create TV series → add episodes → view series budget
- **Fix:** Add Playwright as devDependency, configure `playwright.config.ts`, write 5+ happy-path E2E flows.

### 7. No `data-testid` Attributes
- **Category:** testing
- **Impact:** High (E2E tests impossible without CSS selector coupling)
- **Effort:** M
- **Current state:** 0 `data-testid` attributes across 83 source files.
- **Fix:** Add `data-testid` to all primary interactive elements: nav links, major buttons, form fields, table rows, strip cards. Follow convention `data-testid="[component]-[action]"` (e.g., `data-testid="breakdown-run-button"`).

### 8. SettingsPage Imports 10 Stores (God Component)
- **Category:** engineering
- **Impact:** Medium (maintainability, testing difficulty)
- **Effort:** M
- **Current state:** `SettingsPage.tsx` (759 LOC) imports from 10 different stores. It is the heaviest coupling point in the codebase.
- **Fix:** Extract into logical sub-sections rendered as separate routed views or accordions: `AISettings`, `BudgetSettings`, `ExportSettings`, `AgentSettings`. Each sub-component would import only 1-2 stores.
- **Files:** [src/pages/SettingsPage.tsx](../../src/pages/SettingsPage.tsx)

### 9. BudgetPage and SchedulePage Exceed 1,000 LOC
- **Category:** engineering
- **Impact:** Medium (high cognitive load, hard to review, testing surface too large)
- **Effort:** L
- **Current state:**
  - `BudgetPage.tsx`: 1,196 LOC
  - `SchedulePage.tsx`: 1,179 LOC
  - `AssistantDirectorPanel.tsx`: 1,118 LOC
  - `LineProducerPanel.tsx`: 1,015 LOC
- **Fix:**
  - `BudgetPage`: Extract `BudgetLineTable`, `BudgetDraftHeader`, `BudgetExportPanel`, `InlineNumberEditor` into separate files
  - `SchedulePage`: Extract `StripCard`, `ShootDayRow`, `StripSynopsisPanel`, `InlineEditInput` into separate files
  - AI panels: Extract `AgentChatThread`, `AgentToolCall`, `AgentThinkingIndicator`

### 10. No Code Splitting on Heavy Routes
- **Category:** perf
- **Impact:** Medium (all routes loaded upfront even if user never visits them)
- **Effort:** M
- **Current state:** 0 `React.lazy()` calls. All 17 pages and 8 components are statically imported in `App.tsx`.
- **Fix:** Lazy-load routes with `React.lazy` + `Suspense`:
  - `/project/:id/budget` (BudgetPage: 1,196 LOC)
  - `/project/:id/schedule` (SchedulePage: 1,179 LOC)
  - AI panels (AssistantDirectorPanel, LineProducerPanel, BrainstormPanel — total ~3,700 LOC)
- **Files:** [src/App.tsx](../../src/App.tsx)

### 11. `SeriesNewPage.tsx` Design System Bypass
- **Category:** design
- **Impact:** Low-Medium (raw hex color instead of token)
- **Effort:** S
- **Current state:** `style={{ color: '#E8D500' }}` used for pilot episode heading — bypasses the design system. Should be `className="text-lemon-yellow"`.
- **Files:** [src/pages/SeriesNewPage.tsx](../../src/pages/SeriesNewPage.tsx:211)

### 12. ARIA Density Low — 34 attributes across 133 buttons
- **Category:** a11y
- **Impact:** Medium (screen reader experience degraded in complex pages)
- **Effort:** M
- **Current state:** 34 `aria-*` attributes for 133 button elements + all interactive table cells, strip cards, and form fields. Many interactive elements rely on visual-only context.
- **Fix:** Prioritize `aria-label` on icon-only buttons (export, copy, delete buttons in BudgetPage and SchedulePage), `aria-expanded` on collapsible sections, `aria-live` regions for AI streaming output panels.

## Strengths (Do Not Change)

These are done correctly and should be preserved:

- **TypeScript strictness:** 0 `any` types, `noUncheckedIndexedAccess` enabled. Exemplary.
- **Money math:** All monetary values as integer centavos — no float bugs possible.
- **State architecture:** 11 Zustand stores with clean domain separation and auth-state singleton pattern. No circular imports.
- **CSS discipline:** 1:23.7 inline/className ratio. Tailwind used consistently.
- **Design token system:** 26 well-named CSS custom properties, 3-font system (Barlow Condensed / Archivo / Space Mono) applied uniformly.
- **Responsiveness:** Auth screen adapts flawlessly at all 3 viewports — mobile layout is correctly constrained and readable.
- **Dependency freshness:** All 32 packages on latest stable versions. No stale dependencies.
- **Unit test quality:** All 116 existing tests are behavior-driven with realistic fixtures. High signal-to-noise.
- **Optimistic UI pattern:** All stores update local state immediately before Firestore write — correct pattern for a real-time app.
